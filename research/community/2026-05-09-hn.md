### Notable Discussions

- **AI slop is killing online communities (807 pts, 701 comments)**: Now the dominant AI thread overnight, climbing from 632/557 yesterday. The post argues that auto-generated low-effort content has crossed an inflection point where moderators can't keep up, and the comment thread is a long argument over whether reputation/web-of-trust systems can scale to fix this — [HN](https://news.ycombinator.com/item?id=48053203)
- **Chrome on-device AI not sending data — until they removed the claim (615 pts, 242 comments)**: Google quietly removed the marketing claim that Chrome's on-device AI doesn't transmit data to Google servers. Privacy commenters note this lines up with the rumored shift to a hybrid local/cloud inference path for Gemini Nano — [HN](https://news.ycombinator.com/item?id=48050964)
- **Agents need control flow, not more prompts (574 pts, 285 comments)**: Still climbing from 562/281 yesterday evening. Argues reliable agent systems come from explicit graphs/state machines around the LLM, not larger prompts. Now landing as the top agent-architecture thread of the week — [HN](https://news.ycombinator.com/item?id=48051562)
- **Hardening Firefox with Claude Mythos Preview (344 pts, 154 comments)**: Mozilla published the engineering writeup on using Anthropic's Claude Mythos Preview to find memory-safety issues in Firefox internals. Discussion centers on which classes of bugs the model caught vs. missed and how the harness was built — [HN](https://news.ycombinator.com/item?id=48051079)
- **AI is breaking two vulnerability cultures (300 pts, 125 comments)**: New post from jefftk arguing that AI-driven vuln research is fracturing the disclosure ecosystem — bug-bounty programs are being flooded by LLM-generated reports while serious researchers are getting drowned out. Pairs directly with the Mozilla/Claude Mythos thread — [HN](https://news.ycombinator.com/item?id=48066524)
- **OpenAI's WebRTC problem (218 pts, 58 comments)**: moq.dev's deep-dive on why OpenAI's realtime audio API hits a wall on WebRTC's transport assumptions. Specific to streaming-LLM productization at scale — [HN](https://news.ycombinator.com/item?id=48051951)
- **GPT-5.5 Price Increase (205 pts, 63 comments)**: Up from 186/56. OpenRouter's cost analysis is now the most-referenced pricing post for the GPT-5.5 launch — discussion of whether prompt-caching offsets the per-token bump continues — [HN](https://news.ycombinator.com/item?id=48057209)
- **Teaching Claude Why (123 pts, 55 comments)**: New Anthropic research post on training models to reason about underlying causes vs. surface patterns. Methodology debate is active in comments — [HN](https://news.ycombinator.com/item?id=48066592)
- **ZAYA1-8B matches DeepSeek-R1 on math with <1B active parameters (112 pts, 55 comments)**: Sparse / MoE-based 8B model with sub-1B active parameters claiming DeepSeek-R1 parity on math benchmarks. Reception cautiously positive — most pushback is on benchmark contamination — [HN](https://news.ycombinator.com/item?id=48047082)
- **Principles for agent-native CLIs (109 pts, 49 comments)**: Twitter thread by @trevin on what CLI design changes when the primary user is an LLM agent rather than a human — argument-parsing, error-message verbosity, and idempotency conventions — [HN](https://news.ycombinator.com/item?id=48052333)
- **Show HN: Git for AI Agents — re_gent (97 pts, 46 comments)**: Up from 78/42. Version-control tool with agent-aware commit semantics (rollback by intent, branch-per-task) — comments compare to Aider and Continue's diff-tracking — [HN](https://news.ycombinator.com/item?id=48063548)
- **A recent experience with ChatGPT 5.5 Pro (66 pts, 4 comments)**: Fields Medalist Tim Gowers's blog post on using GPT-5.5 Pro for non-trivial mathematics — fresh, just hitting front page.
