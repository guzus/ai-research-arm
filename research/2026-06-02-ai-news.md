- **The opening framing:** **Satya Nadella** opened the June 2 keynote at **Fort Mason Center** in front of **~5,000 developers and IT pros**, declaring on stage: **"Agents are not just a feature. They are the new operating system for work."** The keynote is structured around four pillars — **Office 365 Copilot, GitHub Copilot, Azure AI Foundry, and Windows Local AI** — with the explicit thesis that Windows is no longer a "platform for human users only" and that agents are first-class citizens in the runtime, tooling and distribution stack.
- **(1) Project Polaris — confirmed as the default GitHub Copilot reasoning engine:**
  - **The model:** **Project Polaris**, the in-house Microsoft AI coding model built by the **MAI** team under **Mustafa Suleyman**, formally unveiled on stage as the future default reasoning engine for GitHub Copilot.
  - **Migration cadence:** Polaris **replaces GPT-4 Turbo as the default for Copilot subscribers starting August 2026**, with **automatic migration** and an **optional three-month GPT-4 fallback** for teams that need it.
  - **Technical differentiator:** **Chain-of-thought + tree-of-thought reasoning at inference time** for complex multi-file refactoring — the workload class where Claude Code has won developer share through 2026 (the strategic anchor that the May 31 carry already named).
  - **Distribution:** Across the new **Copilot Starter (free) / Copilot Pro / Copilot Enterprise** three-tier structure detailed on stage.
- **(2) MAI-Thinking-1 — Microsoft's first dedicated reasoning model:**
  - **The model:** **Mustafa Suleyman** unveiled **MAI-Thinking-1** as the first dedicated reasoning model from Microsoft AI, **built without distillation** (i.e., not trained on outputs from another model — a direct counter-position to the "distillation from frontier-lab models" pattern that dominated 2024–2025 open-source reasoning releases).
  - **Positioning:** Enterprise-focused reasoning model, complementary to Polaris (coding) and to the MAI-Image / MAI-Voice / MAI-Transcribe consumer-modality stack.
  - **Carry-context:** MAI-Image 2.5, MAI-Voice 2 and MAI-Transcribe 1.5 (previewed in yesterday's leak coverage) were also formally unveiled in the same Build segment.
- **(3) Windows Agent Framework (WAF) v1.0 — MIT-licensed at Build:**
  - **The framework:** **Windows Agent Framework v1.0** open-sourced under the **MIT license** — Microsoft's library for building agents that run across **local Windows machines, Windows 365 Cloud PCs, and Azure Arc-enabled edge devices**.
  - **Strategic read:** The MIT license is structurally permissive — competitors and forks can ship WAF-compatible runtimes; the obvious play is to make WAF the *de facto* multi-target agent ABI on Windows the same way .NET became the *de facto* runtime in the 2000s.
- **(4) Windows Local AI — system-level NPU runtime, ships June 9:**
  - **Announced by Windows chief Pavan Davuluri.** **Windows Local AI** is a system-level runtime that lets developers deploy AI agents **running entirely on the neural processing units (NPUs)** inside **Snapdragon X Elite, Intel Lunar Lake, and AMD XDNA-powered PCs**.
  - **Shipping vehicle:** **Windows 11 version 24H2 KB5039239, available June 9, 2026.**
  - **Developer surface:** The **Windows Copilot Runtime API** exposes a **unified ONNX graph across NPU, GPU and CPU** — so a single model artifact can fan out across silicon vendors without re-targeting.
  - **Bundled model:** A **Phi-4-mini-silicon** variant **optimized for Intel, AMD and Qualcomm architectures** ships in-runtime.
  - **Demo:** Davuluri showed a **local Meeting Recap Agent** parsing a Teams transcript stored on-device and generating minutes **in under two seconds — no data leaves the device**. On-stage quote: *"This is for airplane mode, for sensitive data, for latency-critical applications."*
- **(5) Office 365 Copilot Agent Mode — new default for M365 Copilot:**
  - **The change:** Office 365 Copilot **Agent Mode** becomes the new default surface, letting users deploy **persistent AI agents directly inside Word, Excel, Teams and other M365 apps**.
  - **Rollout:** Begins **late June 2026** to all Copilot for Microsoft 365 subscribers.
- **(6) Azure AI Foundry — Agent Orchestrator service:**
  - **Agent Orchestrator** (preview **August 2026**) handles **load balancing across thousands of agents**, integrates with **Azure Logic Apps** for workflow triggers, and provides a **unified billing and usage dashboard**.
- **(7) Microsoft Scout + Copilot "Autopilot" super app — previewed, GA end of summer:**
  - **Scout** is a **proactive agent** (Microsoft's first; previewed in the June 2 keynote) that "watches the user's work and takes initiative, suggesting or launching tasks autonomously" rather than waiting for an explicit prompt.
  - **The super app** folds **Copilot chat, GitHub Copilot, Copilot Cowork** and a new **Autopilot** agentic workflow layer into a single unified Copilot app — internal slogan **"Delivering one Copilot,"** project lead **Jacob Andreou** (recently appointed head of Copilot).
  - **GA target:** **End of summer 2026** (preview only at Build).

(1) **Polaris going from "leak coverage / late-day addition" yesterday to *confirmed on-stage August 2026 default-migration* today is the cleanest single inflection in the Microsoft-vs-Anthropic enterprise-coding contest of 2026.** The August 2026 timing matters: it pulls the Copilot platform off per-token Anthropic pricing exactly when (a) the Microsoft Experiences & Devices team's internal Claude Code license expires (June 30; carry from May 27–31) and (b) Anthropic's own confidential S-1 (filed June 1; carry) brings enterprise-seat retention into the public-market valuation conversation. The migration is **automatic** with **optional three-month GPT-4 fallback** — meaning Microsoft is structurally betting that the GitHub Copilot installed base will not churn out during the swap. Watch for an Anthropic counter-pricing move (a fixed-rate Claude Code Enterprise tier) at **Code with Claude Tokyo (June 10)** or **Q3 London / SF** — the carry-pricing prediction (~$30–60/seat/month with capped token budgets) now has a hard test window. (2) **MAI-Thinking-1's "no distillation" framing is the single most strategic technical claim from the Build keynote.** Through 2024–2025, the dominant open-source reasoning playbook was to distill from frontier closed-weight models (DeepSeek-R1 from Claude, Qwen-QwQ from OpenAI o1, Llama 3.x reasoning from various). MAI-Thinking-1 explicitly opts out — the implicit claim is that Microsoft AI has now trained a reasoning model from scratch at frontier-competitive quality without leaning on Anthropic / OpenAI outputs as supervision. If that claim survives third-party benchmark scrutiny over the next two weeks, it changes the read on **how dependent Microsoft remains on the OpenAI partnership**: not just at the *product* layer (Polaris replaces GPT-4 Turbo in Copilot) but at the *model-research* layer (Microsoft can now train frontier reasoning models in-house). The OpenAI confidential-filing slip (still un-confirmed as of June 1 close; carry) reads differently in that light — OpenAI's partner just stopped needing OpenAI as a teacher. (3) **The Windows Local AI runtime + WAF MIT license + Phi-4-mini-silicon bundle is the structural answer to NVIDIA's Computex N1X / RTX Spark drop yesterday.** NVIDIA's pitch was: *buy our chip and run agentic AI locally*. Microsoft's pitch today is: *use the runtime that's already in every Windows 11 24H2 device, on whichever silicon you already have*. Phi-4-mini-silicon shipping pre-optimized for Snapdragon X Elite, Intel Lunar Lake AND AMD XDNA — without N1X — means the **Windows-on-Arm-via-NVIDIA-N1X 2H 2026 OEM-share story** (yesterday's read) now has a soft floor from Microsoft's own runtime even if N1X devices slip. The two announcements are not mutually exclusive — but the OEM economics tip toward Intel + AMD + Qualcomm if Microsoft's runtime ships first and works well, because the N1X premium is harder to justify when Local AI already runs Phi-4-mini-silicon on existing NPUs. Read alongside the **June 9 KB5039239 shipping date**: Microsoft is shipping the Local AI runtime **two days before SpaceX prices its IPO (June 11) and three days before SPCX opens trading (June 12)** — the cleanest possible competitive pressure on N1X-equipped laptops shipping in fall 2026.

Read against **Microsoft Build's same-week Polaris drop** (above), the strategic picture is: **Microsoft is moving away from OpenAI at the Copilot tier (Polaris August 2026 default), and OpenAI is moving away from Microsoft-exclusive distribution at the cloud tier (AWS GA June 1).** Symmetric strategic divergence; both sides have alternative-stack options now.
