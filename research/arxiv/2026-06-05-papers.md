# arXiv AI Research - 2026-06-05

## Highlights (Top 5 Must-Read)

### 1. When Autoregressive Consistency Hurts Safety Alignment
**Authors**: see [arXiv:2606.04168](https://arxiv.org/abs/2606.04168)
**arXiv**: https://arxiv.org/abs/2606.04168
**TL;DR**: Offers a mechanistic explanation for why safety alignment is "shallow": autoregressive consistency concentrates fine-tuning updates on the first few output tokens, so refusal behavior is anchored early and collapses once a model is nudged past those tokens. The paper shows this is an intrinsic property of next-token training rather than a tuning artifact.
**Why it matters**: Reframes the well-known shallow-safety problem as a structural consequence of the training objective, pointing to objective-level (not just data-level) fixes.

### 2. Inference-Time Vulnerability Beyond Shallow Safety: Alignment Along Generation Trajectories
**Authors**: see [arXiv:2606.04778](https://arxiv.org/abs/2606.04778)
**arXiv**: https://arxiv.org/abs/2606.04778
**TL;DR**: Demonstrates that short token injections at *any* step of generation — not just the prompt — can redirect a safety-aligned model toward harmful output, exposing vulnerability deep in the decoding trajectory. The authors propose aligning models over full generation trajectories rather than only prompt-conditioned first tokens.
**Why it matters**: Extends the shallow-safety literature from "first tokens" to the entire decode path, a more realistic adversarial threat model for deployed assistants.

### 3. CART: Context-Anchored Recurrent Transformer — A Parameter-Efficient Architecture with Learned Stability
**Authors**: see [arXiv:2606.01495](https://arxiv.org/abs/2606.01495)
**arXiv**: https://arxiv.org/abs/2606.01495
**TL;DR**: A looped/recurrent transformer that reuses a single shared core block R times across depth. Unlike prior looped transformers, CART computes K and V once from a multi-layer prelude and has the recurrent core cross-attend to those *frozen* tensors via multi-head latent attention; a learned Linear Time-Invariant gate keeps the recurrence stable (spectral radius settling to ρ ∈ [0.79, 0.83] across all 36 trained configs).
**Why it matters**: A clean parameter-efficient architecture with an explicit, measurable stability guarantee — a rare combination in looped-transformer work.

### 4. Orchestrating Cognitive Memory for Conversational Agents
**Authors**: Meta — see [arXiv:2606.04120](https://arxiv.org/abs/2606.04120)
**arXiv**: https://arxiv.org/abs/2606.04120
**TL;DR**: Trains a language model to manage cognitively-structured memory (user facts, preferences, working memory) for lifelong conversational agents. Argues that naively expanding context with raw retrieval degrades reasoning quality and that vanilla RL hits severe credit-assignment bottlenecks, and proposes a training scheme that sidesteps both.
**Why it matters**: Moves agent memory from "stuff more into the context window" toward a learned, structured memory controller — a likely direction for production assistants.

### 5. PlanarBench: Evaluating LLM Spatial Reasoning via Planar Graph Drawing
**Authors**: see [arXiv:2606.02010](https://arxiv.org/abs/2606.02010)
**arXiv**: https://arxiv.org/abs/2606.02010
**TL;DR**: A memorization-resistant benchmark that asks models to draw planar graphs as ASCII art given only an edge list. Evaluates 91 models on 199 simplest non-isomorphic connected planar graphs, isolating genuine spatial reasoning from pattern recall.
**Why it matters**: Spatial reasoning remains a stubborn LLM weakness; PlanarBench gives a large, leakage-resistant probe across nearly a hundred models.
