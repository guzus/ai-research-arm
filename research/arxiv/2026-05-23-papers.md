## Highlights (Top 5 Must-Read)

### 1. Same Architecture, Different Capacity: Optimizer-Induced Spectral Scaling Laws
**Authors**: see arXiv listing
**arXiv**: https://arxiv.org/abs/2605.21803
**TL;DR**: Demonstrates that an *identical* Transformer architecture realizes
markedly different spectral scaling laws depending only on the optimizer used
to train it. Capacity is therefore not fixed by parameter count and shape — the
optimizer reshapes the weight spectrum and, with it, the effective scaling
exponent.
**Why it matters**: Reframes scaling-law work that treats architecture as the
sole lever, implying optimizer choice belongs *inside* the scaling equation.

### 2. On-Policy Consistency Training Improves LLM Safety with Minimal Capability Degradation
**Authors**: see arXiv listing
**arXiv**: https://arxiv.org/abs/2605.21834
**TL;DR**: Introduces On-Policy Consistency Training (OPCT), an alignment method
evaluated jointly across sycophancy, jailbreaking, and safety-awareness. By
training on the model's own on-policy responses it tightens safety behavior
while leaving general capability largely intact.
**Why it matters**: Directly attacks the safety-vs-utility tax that has dogged
alignment fine-tuning, offering a recipe that does not lobotomize the model.

### 3. ACC: Compiling Agent Trajectories for Long-Context Training
**Authors**: see arXiv listing
**arXiv**: https://arxiv.org/abs/2605.21850
**TL;DR**: Agents emit massive multi-turn trajectories with the evidence needed
to answer scattered across distant turns. ACC "compiles" those raw trajectories
into compact long-context training data so models learn to integrate dispersed
context rather than drowning in it.
**Why it matters**: Turns the exhaust of agent runs into a reusable training
signal — a scalable path to better long-horizon, long-context agents.

### 4. Vision-OPD: Learning to See Fine Details for Multimodal LLMs
**Authors**: see arXiv listing
**arXiv**: https://arxiv.org/abs/2605.18740
**TL;DR**: Targets the persistent failure of multimodal LLMs at fine-grained
visual understanding, adding an object/part-detail mechanism so the model
attends to small but decision-critical image regions instead of coarse global
features.
**Why it matters**: Fine-detail perception is the bottleneck for MLLMs in
document, chart, and inspection tasks; closing it widens real-world deployment.

### 5. Continuous Latent Diffusion Language Model (Cola DLM)
**Authors**: see arXiv listing
**arXiv**: https://arxiv.org/abs/2605.06548
**TL;DR**: A hierarchical latent diffusion language model that generates text
through hierarchical information decomposition — a Text VAE learns a stable
text-to-latent map while a global semantic prior is modeled in continuous
latent space, rather than autoregressive token-by-token decoding.
**Why it matters**: A credible non-autoregressive architecture for text
generation, advancing the diffusion-LM line as an alternative to AR decoding.
