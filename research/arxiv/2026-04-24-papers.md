# arXiv AI Research - 2026-04-24

## Highlights (Top 5 Must-Read)

### 1. CoEvolve: Training LLM Agents via Agent-Data Mutual Evolution
**Authors**: Multiple (see arXiv)
**arXiv**: https://arxiv.org/abs/2604.15840
**TL;DR**: Proposes an agent-data mutual evolution framework in which LLM agents and their training data co-evolve through closed-loop, interaction-driven training, rather than relying on a fixed corpus. The agent generates data by acting, and the data set in turn reshapes the agent's capabilities.
**Why it matters**: Moves past static supervised / RL pipelines toward self-improving agents, potentially reducing dependence on human-curated training sets for agent skill acquisition.

### 2. Freshness-Aware Prioritized Experience Replay for LLM/VLM Reinforcement Learning
**Authors**: Multiple (see arXiv)
**arXiv**: https://arxiv.org/abs/2604.16918
**TL;DR**: First work to successfully apply Prioritized Experience Replay to LLM/VLM RL, identifying "priority staleness" as the key failure mode. Introduces a freshness-aware age-decay scheme grounded in importance sampling theory.
**Why it matters**: Addresses a long-standing practical obstacle to sample-efficient RL training of LLMs/VLMs, where replay buffers were previously considered unstable.

### 3. Bringing Value Models Back: Generative Critics for Value Modeling in LLM Reinforcement Learning
**Authors**: Multiple (see arXiv)
**arXiv**: https://arxiv.org/abs/2604.10701
**TL;DR**: Introduces GenAC (Generative Actor-Critic), which replaces one-shot scalar value prediction with a generative critic that performs chain-of-thought reasoning before emitting a value estimate. Revives value-model-based RL for LLMs in the post-PPO / post-DPO era.
**Why it matters**: Suggests a path back to actor-critic RL — previously displaced by preference-based methods — with richer, more interpretable critics.

### 4. KnowRL: Boosting LLM Reasoning via Reinforcement Learning with Minimal-Sufficient Knowledge Guidance
**Authors**: Multiple (see arXiv)
**arXiv**: https://arxiv.org/abs/2604.12627
**TL;DR**: Treats hint design for reasoning RL as a minimal-sufficient guidance problem, using Constrained Subset Search to build compact, interaction-aware knowledge subsets that guide rollouts during training.
**Why it matters**: Offers a principled alternative to dense supervision or pure verifier rewards, pushing the frontier of data-efficient reasoning RL.

### 5. Uni-SafeBench: A Safety Benchmark for Unified Multimodal Large Models
**Authors**: Multiple (see arXiv)
**arXiv**: https://arxiv.org/list/cs.AI/current
**TL;DR**: A benchmark specifically targeting safety of unified multimodal models (understanding + generation in one stack), asking whether unification exposes new attack surfaces absent in separate pipelines.
**Why it matters**: As frontier labs converge on unified MM architectures, systematic red-teaming of the unified stack becomes critical for deployment.
