# arXiv AI Research - 2026-05-14

## Highlights (Top 5 Must-Read)

### 1. Rethinking RL for LLM Reasoning: It's Sparse Policy Selection, Not Capability Learning
**Authors**: (arXiv 2605.06241)
**arXiv**: https://arxiv.org/abs/2605.06241
**TL;DR**: Argues that RL fine-tuning does not teach models new reasoning abilities — instead it reframes reasoning improvement as a sparse policy-selection problem, where the model already possesses the needed reasoning paths and RL only nudges it to commit to the right branch at critical decision points.
**Why it matters**: Reshapes how the field interprets RL gains on reasoning benchmarks and suggests far cheaper training recipes.

### 2. Does RL Expand the Capability Boundary of LLM Agents? A PASS@(k,T) Analysis
**Authors**: (arXiv 2604.14877)
**arXiv**: https://arxiv.org/abs/2604.14877
**TL;DR**: Introduces a PASS@(k,T) metric that jointly varies sample budget k and tool-call horizon T, and shows tool-use RL genuinely enlarges the capability boundary — the RL agent's pass-curve pulls above the base model's, with the gap widening at large k.
**Why it matters**: Provides a rigorous answer to the contested question of whether RL adds capability or just reweights existing behavior — for agents, it does add capability.

### 3. Workspace-Bench 1.0: Benchmarking AI Agents on Workspace Tasks with Large-Scale File Dependencies
**Authors**: (arXiv 2605.03596)
**arXiv**: https://arxiv.org/abs/2605.03596
**TL;DR**: A realistic agent benchmark with 5 worker profiles, 74 file types, 20,476 files (up to 20GB) and 388 tasks, each with its own file dependency graph and evaluated across 7,399 rubrics. State-of-the-art agents average only a 47.4% Rubrics Pass Rate; the best pairing reaches ~70%.
**Why it matters**: Exposes a large gap between frontier agents and genuine multi-file "workspace learning," setting a hard target for agentic systems.

### 4. Reinforcement Learning for LLM-based Multi-Agent Systems through Orchestration Traces
**Authors**: (arXiv 2605.02801)
**arXiv**: https://arxiv.org/abs/2605.02801
**TL;DR**: Models multi-agent systems as temporal interaction graphs (orchestration traces) covering sub-agent spawning, delegation, communication, tool use, aggregation and stopping, and trains RL over those traces — including the previously unaddressed stopping decision.
**Why it matters**: First end-to-end RL formulation for orchestrated multi-agent systems, a key missing piece for reliable agent pipelines.

### 5. Hardware Co-Design Scaling Laws via Roofline Modelling for On-Device LLMs
**Authors**: (arXiv 2602.10377)
**arXiv**: https://arxiv.org/abs/2602.10377
**TL;DR**: Proposes a co-design scaling law that models training loss as an explicit function of architectural hyperparameters while characterizing inference latency via roofline modelling. At equal latency to Qwen2.5-0.5B on target hardware, the co-designed architecture achieves 19.42% lower WikiText-2 perplexity.
**Why it matters**: Bridges scaling-law theory and real hardware constraints, directly useful for edge/on-device deployment.
