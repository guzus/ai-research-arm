# arXiv AI Research - 2026-01-31

## Highlights (Top 5 Must-Read)

### 1. [Keel: Post-LayerNorm Is Back — Stable, Expressive, and Deep](https://arxiv.org/abs/2601.19895)
**Authors**: Chen Chen, Lai Wei (ByteDance Seed)
**arXiv**: [2601.19895](https://arxiv.org/abs/2601.19895)
**TL;DR**: Introduces Keel, a Post-LayerNorm Transformer that replaces ResNet-style residual paths with Highway-style connections, enabling stable training at 1000+ layers. Keel consistently outperforms Pre-LN across all depths (64–1024 layers), with +16.5% on Math & Code benchmarks.
**Why it matters**: Reopens depth as a viable scaling dimension for LLMs — a potential paradigm shift away from width-only scaling, with implications for next-generation architecture design.

### 2. [Recursive Language Models](https://arxiv.org/abs/2512.24601)
**Authors**: Alex L. Zhang, Tim Kraska, Omar Khattab (MIT OASYS)
**arXiv**: [2512.24601](https://arxiv.org/abs/2512.24601) (v2 updated Jan 28, 2026)
**TL;DR**: Proposes RLMs, a paradigm where the LLM never sees long prompts directly but interacts with context via a Python REPL, recursively spawning sub-LLMs. RLM-Qwen3-8B outperforms base Qwen3-8B by 28.3% and approaches GPT-5 quality on long-context tasks while handling inputs 100x beyond context windows.
**Why it matters**: Reframes context length as an environment management problem rather than an architecture problem — Prime Intellect calls this "the paradigm of 2026."

### 3. [Breaking the Overscaling Curse: Thinking Parallelism Before Parallel Thinking](https://arxiv.org/abs/2601.21619)
**Authors**: Yiming Wang, Zhuosheng Zhang, Rui Wang
**arXiv**: [2601.21619](https://arxiv.org/abs/2601.21619)
**TL;DR**: Identifies the "overscaling curse" in parallel thinking (multi-path sampling): system-level gains come at sample-level inefficiency. Proposes T2, which uses latent representations to estimate optimal parallelism per sample before decoding, reducing cost by up to 60% while matching performance.
**Why it matters**: First principled solution to the cost-performance tradeoff of test-time scaling via parallel reasoning, critical for practical deployment of reasoning models.

### 4. [A Model of Errors in Transformers](https://arxiv.org/abs/2601.14175)
**Authors**: Suvrat Raju, Praneeth Netrapalli
**arXiv**: [2601.14175](https://arxiv.org/abs/2601.14175)
**TL;DR**: Derives a two-parameter "effective field theory" model of transformer error rates on deterministic tasks (arithmetic, dynamic programming, Hanoi). Shows errors arise from accumulated attention noise crossing a threshold. Validated extensively on Gemini 2.5 Flash/Pro and DeepSeek R1.
**Why it matters**: Provides the first quantitative predictive model for when and why LLMs make errors on structured tasks — offers principled methods to reduce error rates via prompt design.

### 5. [From Meta-Thought to Execution: Cognitively Aligned Post-Training for Generalizable LLM Reasoning](https://arxiv.org/abs/2601.21909)
**Authors**: Shaojie Wang et al.
**arXiv**: [2601.21909](https://arxiv.org/abs/2601.21909)
**TL;DR**: Proposes Chain-of-Meta-Thought (CoMT), a cognitively-inspired framework that separates abstract reasoning pattern learning (via SFT) from specific execution (via RL). Models learn generalizable strategies rather than task-specific solutions, improving transfer across reasoning domains.
**Why it matters**: Bridges cognitive science and LLM training — addresses the fundamental limitation that current post-training optimizes for specific trajectories rather than transferable reasoning skills.
