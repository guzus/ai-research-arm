# arXiv AI Research - 2026-06-16

## Highlights (Top 5 Must-Read)

### 1. ARM: An AutoRegressive Large Multimodal Model with Unified Discrete Representations
**Authors**: (arXiv:2606.11188)
**arXiv**: https://arxiv.org/abs/2606.11188
**TL;DR**: A 7B discrete-representation autoregressive model that unifies image understanding, generation, and editing in a single architecture. By mapping vision and language into a shared discrete token space, one decoder-only model handles perception and generation without separate task heads.
**Why it matters**: Pushes the "single autoregressive backbone for everything" thesis into a genuinely unified perceive-and-generate multimodal model, a direction the frontier labs are racing toward.

### 2. WorldReasoner: Evaluating Whether Language Model Agents Forecast Events with Valid Reasoning
**Authors**: (arXiv:2606.11816)
**arXiv**: https://arxiv.org/abs/2606.11816
**TL;DR**: An evaluation framework for temporally valid event forecasting: agents are given resolved forecasting questions plus access only to evidence available before a simulated forecast date, isolating reasoning quality from leakage. It separates "got the answer right" from "reasoned validly."
**Why it matters**: Forecasting benchmarks routinely leak future information; WorldReasoner builds the discipline of testing whether agents reason soundly, not just luckily.

### 3. Back on Track: Aligning Rewards and States for Reasoning in Diffusion Large Language Models
**Authors**: (arXiv:2606.08501)
**arXiv**: https://arxiv.org/abs/2606.08501
**TL;DR**: An RL method that enhances reasoning in diffusion LLMs (dLLMs) by aligning rewards with intermediate denoising states, addressing the mismatch between diffusion's non-autoregressive generation and standard RL reward assignment.
**Why it matters**: Diffusion LLMs are the leading non-autoregressive alternative; making RL-for-reasoning work on them is a key step toward competitive dLLM reasoners.

### 4. Grammar-Constrained Decoding Can Jailbreak LLMs into Generating Malicious Code
**Authors**: (arXiv:2606.11817)
**arXiv**: https://arxiv.org/abs/2606.11817
**TL;DR**: Introduces CodeSpear, a jailbreak that exploits grammar-constrained decoding (GCD) to coerce LLMs into emitting malicious code, achieving ~81.8% average attack success against locally deployed models. Proposes CodeShield, an alignment defense that survives attacker-controlled grammar constraints.
**Why it matters**: GCD is widely used for structured/tool output — this exposes it as a safety bypass vector and offers a concrete mitigation.

### 5. Scaling Laws for Behavioral Foundation Models over User Event Sequences
**Authors**: (arXiv:2606.05257)
**arXiv**: https://arxiv.org/abs/2606.05257
**TL;DR**: Across ~600 runs spanning 10^15–10^19 FLOPs, studies a feature-embedder + decoder-only transformer for next-event prediction over user behavior. Finds a small embedder (~2% of params) is compute-optimal at every budget tested.
**Why it matters**: Extends Chinchilla-style scaling-law methodology to behavioral/sequence foundation models and gives a concrete compute-optimal allocation rule.
