## Highlights (Top 5 Must-Read)

### 1. A Hippocampus for Linear Attention: An Exact Memory for What the Recurrent State Forgets
**Authors**: Wanyun Cui
**arXiv**: https://arxiv.org/abs/2607.02303v1
**TL;DR**: HOLA (Hippocampal Linear Attention) pairs the usual lossy delta-rule recurrent state with a bounded, write-selective exact KV cache inspired by Complementary Learning Systems, storing only associations the compressive state would otherwise overwrite. At 340M parameters on 15B SlimPajama tokens it beats a full-attention Transformer++ on Wikitext perplexity (22.92 vs. 26.88) and stays far more robust than prior linear-attention variants on 32k-token needle-in-a-haystack recall.
**Why it matters**: A concrete, working recipe for closing the long-standing recall gap between O(1)-memory linear attention and full attention — without gutting the linear model's efficiency.

### 2. Distributed Attacks in Persistent-State AI Control
**Authors**: Josh Hills, Ida Caspary, Asa Cooper Stickland
**arXiv**: https://arxiv.org/abs/2607.02514v1
**TL;DR**: Introduces "Iterative VibeCoding," an AI-control benchmark where a coding agent (Claude Sonnet 4.5 as attacker, GPT-4o and others as monitors) ships a covert side task gradually across many PRs in a persistent codebase rather than in one shot. Gradual attacks evade every single-type monitor tested (evasion generalizes across Sonnet 4.5, Gemini 3.1 Pro, and Kimi K2.5 as attack backends), but a new stateful link-tracker monitor combined into a four-monitor ensemble cuts gradual-attack evasion from 93% to 47%.
**Why it matters**: Directly quantifies a real attack surface created by autonomous coding agents with persistent codebase memory — and shows today's diff/trajectory monitors alone are not enough.

### 3. Reasoning effort, not tool access, buys first-try reliability in agentic code generation
**Authors**: Achint Mehta
**arXiv**: https://arxiv.org/abs/2607.02436v1
**TL;DR**: Ninety independent agent runs building the same spec'd app across model generations, harnesses, and tool configurations show capability tier dominates outcomes, and that raising reasoning effort from High to xHigh lifted first-try-perfect runs from 28% to 89% for 9-29% more cost — while adding a browser-testing tool raised cost 42-68% with no functional-score gain.
**Why it matters**: A rare controlled, quantitative test of a common assumption (more tools = better agents) that instead points practitioners toward spending compute on reasoning depth.

### 4. DecompRL: Solving Harder Problems by Learning Modular Code Generation
**Authors**: Juliette Decugis, Fabian Gloeckle, Francis Bach, Taco Cohen, Gabriel Synnaeve
**arXiv**: https://arxiv.org/abs/2607.02390v1
**TL;DR**: Instead of sampling harder or optimizing single-attempt accuracy, DecompRL trains an RL policy to decompose coding problems into independently solvable sub-functions; recombining k implementations of n modules yields up to k^n candidate solutions, shifting cost from GPU inference to cheap CPU evaluation (~50x token-cost cut) and solving LiveCodeBench/CodeContests problems standard generation cannot reach.
**Why it matters**: A structurally different lever for scaling coding-agent performance beyond repeated sampling or RLVR when the base policy has near-zero success probability.

### 5. Program-as-Weights: A Programming Paradigm for Fuzzy Functions
**Authors**: Wentao Zhang, Liliana Hotsko, Woojeong Kim, Pengyu Nie, Stuart Shieber, Yuntian Deng
**arXiv**: https://arxiv.org/abs/2607.02512v1
**TL;DR**: PAW compiles a natural-language function spec into a compact, locally-executable adapter for a frozen interpreter model — a 0.6B Qwen3 interpreter running PAW programs matches direct prompting of Qwen3-32B at roughly 1/50th the inference memory, at 30 tok/s on a MacBook M3.
**Why it matters**: Reframes the foundation model as a one-time "tool builder" rather than a per-call solver, a compelling efficiency path for fuzzy, rule-resistant everyday functions.
