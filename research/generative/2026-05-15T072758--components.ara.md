This document is its own integration test. It exercises every primitive at least once. If a primitive renders wrong here, it renders wrong everywhere.

## 01. Headings and structure

Use `ara-display` for the article title (one per doc). Use `ara-h2` for sections (with an optional `ara-h2-num` chip for numbered sections). `ara-h3` and `ara-h4` step down from there.

### An `ara-h3` subsection

Subsections sit beneath h2-level sections. Lighter weight, smaller size.

#### An `ara-h4` minor head

Minor heads are for tight subdivisions inside a subsection. Same family, smaller and muted.

---

## 02. Callouts

Four variants. Use sparingly — every callout costs reader attention.

:::callout(kind=info, label=Note)
Informational callouts use the warm accent color. Good for footnotes, asides, methodology disclosures.
:::

:::callout(kind=success, label=Confirmed)
Green for verified facts, completed actions, positive signals. Use when something is genuinely settled, not just nice.
:::

:::callout(kind=warn, label=Caveat)
Yellow for partial confidence, unverified claims, things to watch. Most analytical hedges land here.
:::

:::callout(kind=danger, label=Risk)
Red for red flags. Don't dilute by using it for mere uncertainty — that's what warn is for.
:::

## 03. Pull quote

For a single sentence worth slowing down for. Don't use for normal citations — that's what links are for.

> The bottleneck is no longer compute. It's the human attention required to know what to compute on.
> — Composite take, AI research bench, mid-2026

## 04. Key/value pairs

Definition lists with `ara-kv`. Good for "at a glance" metadata — company facts, paper details, contract terms.

:::kv
- term: Founded
  def: '1998'
- term: HQ
  def: Cambridge, MA
- term: Employees
  def: ~10,400
- term: Public ticker
  def: AKAM (Nasdaq)
:::

## 05. Stat grid

Headline numbers. Tabular-figure mono ensures decimals line up. Auto-fits 140px+ columns.

  value: '59.3%'
  note: stable
- label: FCF
  value: $1.1B
  note: guidance lifted

:::

## 07. Lists, links, inline elements

Standard unordered list:

- A claim with an [inline source](https://example.com/source).
- Another claim with *emphasis*, **strong emphasis**, and `inline code`.
- A statement followed by a numbered citation[^1].

Ordered list:

1. First step in a sequence.
2. Second step.
3. Third step.

## 08. Tags, flags, highlights

Inline tags: {tag}cdn{/} {tag}security{/} {tag}edge-compute{/}

Status flags inline: {flag:green}solid; {flag:yellow}watch; {flag:red}concerning.

Highlighted text: ==key insight== draws the eye.

## 09. Bars — single-value comparison

For when "`$520M`" in a stat block isn't enough — you want the reader to *see* the proportion. Each `ara-bar` declares its fill via `data-pct`; the dashboard sets the CSS variable at render time so authors don't touch inline style.

:::bars
- label: Security
  value: $520M · 62%
  pct: 62
- label: Delivery
  value: $380M · 45%
  pct: 45
- label: Compute
  value: $180M · 22%
  pct: 22
:::

## 10. Stacked bar — segmented proportions

One horizontal bar split by category, with a legend underneath. Four neutral fill shades (`ara-stack-seg--1` through `--4`) keep it monochromatic.

:::stack-bar(legend)
- pct: 42
  label: Security 42%
- pct: 28
  label: Delivery 28%
- pct: 18
  label: Compute 18%
- pct: 12
  label: Other 12%
:::

## 11. Timeline — vertical event log

For chronological lists where dates carry semantic weight. Use it for funding rounds, product launches, incident timelines, governance events.

:::timeline
- date: '2024-01'
  headline: Series B raised
  body: $150M at a $1.2B valuation, led by Lightspeed.
- date: '2024-06'
  headline: Product launch
  body: Enterprise tier opened to design partners.
- date: '2025-02'
  headline: First major outage
  body: Three-hour edge-cache invalidation hit ~30% of customers.
:::

## 12. Compare trio — min, max, subject

Bloomberg-style three-card block for "where does the subject sit on the distribution." Use for any "lowest / highest / you" comparison. Mark the subject's card with `ara-compare-card--subject` to tint it.

:::compare
- role: Lowest
  name: Italy
  value: '4%'
- role: Highest
  name: South Korea
  value: '27%'
- role: United States
  name: '2026 cohort'
  value: '25%'
:::

## 14. Pictogram count — isotype-style ratio

For ratio-at-a-glance comparisons: pick a glyph, set `data-count`, and the dashboard expands it into repeated child spans at render time. Cap is 200. Pair two rows for "then vs. now."

:::iso
- label: '1950'
  glyph: 🚶
  count: 17
- label: '2010'
  glyph: 🚶
  count: 3
:::

## 15. Inline accent + unit suffix

Use `ara-accent` to anchor the eye on "the subject" across a global comparison — distinct from `<em>` (italic) and `ara-mark` (background tint). Use `ara-stat-unit` for tiny unit suffixes that hang off big figures like `87 yrs.` or `$39.6T`.

Italy contributes 45.30% of wages to mandatory social-security programs; {accent}the United States{/} contributes 16.90% — the 17th-highest rate of the 20 countries surveyed.

:::stats
- label: Life expectancy at 60
  value: '87'
  unit: yrs.
  note: U.S. women cohort
- label: Medicare shortfall
  value: $39.6
  unit: T
  note: through 2087
- label: Stargate site
  value: '1.2'
  unit: GW
  note: Abilene, TX
:::

## 17. Sparkline — inline trend

For showing a trend shape inside prose, no axes needed. Wheat futures are up sharply over the last six months {sparkline:5.20,5.40,5.55,5.30,5.10,5.20,5.35,5.50,5.80,6.10,6.40,6.85,7.05} — a level not seen since mid-2024. The dashboard reads `data-points` and stamps an SVG path at render time.

## 18. Line chart — multi-series with axes

When the trend needs axes, gridlines, and possibly multiple series side-by-side, use `ara-line-chart`. Up to four series via `data-series-1` through `data-series-4`.

:::

## 20. Slopegraph — two-period change

For showing how a set of items changed between two periods. Lines that went up render green; lines that went down render red. Best when ≤8 items.

:::slope(left-label="2020 mkt cap", right-label="2026 mkt cap", unit=$)
| Item | 2020 mkt cap | 2026 mkt cap |
|---|---|---|
| Apple | 2.0 | 3.4 |
| Microsoft | 1.6 | 3.1 |
| Nvidia | 0.4 | 3.0 |
| Alphabet | 1.4 | 2.4 |
| Amazon | 1.0 | 1.9 |
| Meta | 0.8 | 1.6 |
:::

## 21. Code blocks

Inline `code` and fenced blocks:

```
def compose(article: str) -> bool:
    # Every class must start with ara-
    return all(c.startswith("ara-") for c in classes_in(article))
```

## 22. What's NOT here

Components that don't exist by design:

- **No layout primitives.** No `ara-grid`, `ara-row`, `ara-col`. Articles flow top-to-bottom; if you need a grid, it's probably `ara-stats` or a table.
- **No color utilities.** No `ara-text-red`. Use a callout or a flag for semantic color.
- **No spacing utilities.** Margins are baked into each component. If two components feel cramped, the component's own margins should be edited.
- **No theming knobs.** Light/dark come from the dashboard's CSS variables. Articles never pick colors directly.
