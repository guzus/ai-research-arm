## Changes in this PR

### Committed directly (no `workflows: write` needed)

1. **`scripts/check-feeds.sh`** — a standalone health-check script that
   probes every RSS and Bluesky endpoint in the pipeline and prints a
   one-row-per-feed table of HTTP status, content-shape, and freshness
   timestamp. Run with `bash scripts/check-feeds.sh`. Exits non-zero if
   any single feed fails, so it can be wired into CI or a separate
   workflow later. The known-broken legacy URLs are included so they
   appear in the output as `FAIL` rows — making the next "feed silently
   rotted" incident visible within one minute instead of seventeen days.

2. **`improve-2026-05-14-workflow-fixes.patch`** — a `git apply`-able
   patch that fixes the two workflow files. Differences vs the
   `improve/2026-05-13` patch:

     with the canonical `research.google/blog/rss/`.
