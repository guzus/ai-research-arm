## Issues Identified

### 1. Bluesky pipeline silently broken since 2026-04-28 (HIGH)

`research/bluesky/2026-04-28.md` through `2026-05-03.md` all log:

> _No posts processed: upstream Bluesky fetch returned HTTP 403 Forbidden
> for all source feeds (ai_announcements, llm_releases, model_discussions,
> ml_papers, karpathy)._

**Root cause** (re-confirmed by `curl -I` from this runner):
`public.api.bsky.app` started returning HTTP 403 with a BunnyCDN HTML error
page for unauthenticated XRPC calls — a CDN/anti-bot block on the
runner's datacenter IP range. The alternate host `api.bsky.app` exposes
the same XRPC surface on a different CDN and continues to return valid
JSON (verified `HTTP/2 200` + 25KB JSON envelope at run time).

A fix was proposed on `improve/2026-05-02` but only as a patch file
(`IMPROVEMENT-PATCH-2026-05-02.patch`); the workflow itself was never
updated. This PR finally applies the change.

### 2. Hourly RSS workflow has produced zero commits since 2026-04-27 09:18 UTC (HIGH)

`git log` confirms the last `RSS update …` commit was 2026-04-27 09:18 UTC
— **6+ days of complete silence on the official-blog firehose.** Even on
ultra-quiet days the workflow normally produces ~24 hourly "no new
updates" appends; this is structural failure, not signal absence.
`research/rss/` ends at `2026-04-27.md`; April 28–May 3 are missing entirely.

The most likely contributor is **coverage thinness — too few feeds, all
slow-cadence corporate blogs** — leaving the Claude step nothing to
write and producing turn-after-turn of identical "No new updates" output
that may be hitting model-side guardrails or simply timing out the
concurrency window. We address this by broadening the source set so
the workflow has substantive content to extract on most hours.

### 3. `daily-improve.yml` has been creating branches but no PRs (MEDIUM)

`origin/improve/2026-04-28` through `origin/improve/2026-05-02` all exist
with commits, but no PRs were opened against any of them. Cause: the
prompt instructs `gh pr create` but the `allowedTools` list is
`Read,Write,Edit,Bash(git:*),WebSearch` — `Bash(gh:*)` is missing, and
`GH_TOKEN` is never exported, so `gh pr create` cannot run.

This means the entire self-improvement loop has been losing every output
for almost a week. Without PRs, no human reviewer sees the proposed
diffs, and they never reach `main`.

## Changes Made

### `.github/workflows/2h-bluesky.yml` — Bluesky 403 fix

1. Switch host `public.api.bsky.app` → `api.bsky.app`.
2. Replace the broken `searchPosts?q=from:user.bsky.social` form (Bluesky
   has no `from:` operator — it silently returned zero results) with
   real `getAuthorFeed` calls for **5 high-signal AI researchers**:
   `karpathy`, `rasbt` (Sebastian Raschka), `simonwillison`, `emollick`
   (Ethan Mollick), `sama`.
3. Normalize HTML error pages (responses starting with `<`) to a valid
   empty `{"posts":[]}` / `{"feed":[]}` envelope so a future CDN
   regression degrades gracefully rather than crashing the Claude step
   on HTML-as-JSON.
4. Add `curl --retry 3` + a first-byte audit log so transient 5xx and
   future regressions don't silently produce empty source files.
5. Update the Claude prompt to handle **both response shapes** —
   `searchPosts` returns top-level `posts[]`; `getAuthorFeed` returns
   `feed[].post`.

### `.github/workflows/daily-improve.yml` — make the self-improvement loop actually open PRs

1. Add `GH_TOKEN` to the action's `env:` so `gh` can authenticate.
2. Extend `allowedTools` with `Bash(gh:*)` plus a few read-only Bash
   primitives the analysis step needs (`curl`, `ls`, `wc`, `head`,
   `tail`, `grep`).
3. **Backstop PR creation in the post-action step**: if the Claude step
   left commits on an `improve/<date>` branch but did not create a PR,
   the runner now opens one itself via `gh pr create` (idempotent — skips
   if a PR already exists for the branch). This recovers the
   self-improvement loop even if a future allowedTools regression
   happens.
