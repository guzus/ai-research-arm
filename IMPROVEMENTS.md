## Issues Found & Fixes Applied

### 1. Bluesky API Still Using Wrong URL (HIGH)
**Problem:** Despite the Feb 1 improvement adding failover logic, the Bluesky workflow still used `public.api.bsky.app` as the primary URL, which requires JWT auth. The 10:51 UTC cycle explicitly logged 403/AuthMissing errors, and data collection stopped.

**Fix:** Replaced ALL `public.api.bsky.app` URLs with `api.bsky.app` (the known working endpoint). Removed the failover complexity — using the correct URL directly is more reliable.

### 2. Missing RSS Feeds from Key Publications (MEDIUM)
**Problem:** Ars Technica, Wired, and IEEE Spectrum have confirmed working AI RSS feeds but weren't included. These are high-quality sources that provide analysis beyond what TechCrunch/Verge/VentureBeat cover.

**Fix:** Added 3 new RSS feeds:
- Ars Technica AI (`arstechnica.com/ai/feed/`)
- Wired AI (`wired.com/feed/tag/ai/latest/rss`)
- IEEE Spectrum AI (`spectrum.ieee.org/feeds/topic/artificial-intelligence.rss`)

### 3. Missing arXiv CS.CL Category (MEDIUM)
**Problem:** Only CS.AI and CS.LG were tracked. CS.CL (Computation and Language) is where most LLM papers are first posted — a significant gap for an AI research pipeline.

**Fix:** Added CS.CL to the RSS workflow fetch and processing pipeline.

### 4. Missing Twitter Accounts (MEDIUM)

### 5. Limited Bluesky Search Coverage (LOW)
