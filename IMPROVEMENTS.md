## Issues Found in Yesterday's Output (2026-05-11)

### 1. Bluesky pipeline returning 100% empty results

`research/bluesky/2026-05-11.md` opens with:

> _Bluesky data fetch returned HTTP 403 Forbidden for all source feeds
> (ai_announcements, llm_releases, model_discussions, ml_papers, karpathy).
> No posts available for this run._

Live verification from the runner confirms the cause:

| Endpoint | Response |
|----------|----------|
| `https://public.api.bsky.app/xrpc/app.bsky.feed.searchPosts` | **403 Forbidden** (HTML body) |
| `https://api.bsky.app/xrpc/app.bsky.feed.searchPosts` | **200 OK** with real posts |
| `https://bsky.social/xrpc/app.bsky.feed.searchPosts` | 401 (auth required) |

Bluesky restricted unauthenticated access on the `public.api.bsky.app`
subdomain at some point during 2026 (see
[bluesky-social/bsky-docs#332](https://github.com/bluesky-social/bsky-docs/issues/332)).
The same endpoint on `api.bsky.app` still responds without auth.

## Changes in This PR

> **Note on delivery format:** The GitHub App that runs this daily-improve
> workflow does not hold the `workflows: write` permission, so it cannot
> push edits to `.github/workflows/*.yml` directly. The full diff is
> shipped as **`improvements-2026-05-12-workflows.patch`** in this PR.
> A maintainer can apply it locally with:
>
> ```bash
> git checkout improve/2026-05-12
> git apply improvements-2026-05-12-workflows.patch
> git add .github/workflows/2h-bluesky.yml .github/workflows/hourly-rss.yml
> git commit -m "Apply 2026-05-12 workflow improvements"
> git push
> ```

### `.github/workflows/2h-bluesky.yml`

- Switched `public.api.bsky.app` → `api.bsky.app` (the only verified-working
  unauth endpoint as of 2026-05-12). This is the single change that
  unblocks Bluesky ingestion end-to-end.
- Replaced 5 ad-hoc `curl` lines with a `fetch_bsky` helper that records
  HTTP status and substitutes `{"posts":[]}` only on non-200, so a
  partial outage degrades gracefully instead of all-or-nothing.
- Added file-size summary to job logs so future runs surface per-query
  failures immediately.

### `.github/workflows/hourly-rss.yml`

- Removed broken `https://ai.meta.com/blog/rss/` (404), replaced with
  `https://openrss.org/ai.meta.com/blog` bridge.
- Replaced `http://googleaiblog.blogspot.com/atom.xml` (301 to stale
  content) with `https://research.google/blog/rss/` which covers the
  same Google/DeepMind material at the current canonical URL.
- Added per-feed status check (`fetch_feed` helper): logs HTTP code and
  byte count, only stubs to empty on non-200 or sub-100-byte responses.
  This is the missing observability that allowed a 15-day silent outage.
- Added 7 new feeds, all live-verified 200 from the runner:
  - **Practitioner commentary**: Simon Willison, Latent Space, Import
    AI (Jack Clark) — feeds widely cited inside the AI engineering
    community that historically break or first-analyze frontier-model
    news.
  - **Tech press depth**: Ars Technica AI, MIT Technology Review AI.
  - **Lab coverage**: Mistral, Cohere — fills a gap between
    OpenAI/Anthropic/Google/Meta and the rest of the frontier-lab pack.
- Extended the Claude prompt and output schema to include the new
  categories so additions actually surface in `research/rss/*.md`.
