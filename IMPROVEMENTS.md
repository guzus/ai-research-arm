## Why this keeps recurring

Today is the **fifth consecutive `daily-improve` PR** to identify the
same two silently-broken feeds:

- `improve/2026-05-12`, `2026-05-13`, `2026-05-14`, today — all carry
  effectively the same diagnosis and the same fix.
- No `improve/2026-05-15` branch exists; yesterday's `daily-improve` run
  appears to have produced nothing (or failed before commit).
- The root blocker is the same each day: the `daily-improve` GitHub App
  token lacks `workflows: write`, so the bot can prepare workflow-file
  edits but the closing `git push` fails the moment any changed file is
  under `.github/workflows/`. Net result: 17→18→19 days of zero RSS.

**Two ways to break the loop**:

1. **Grant the `daily-improve` App `workflows: write`** (in the GitHub
   App's repository permissions, plus `actions: write` on the workflow's
   `permissions:` block). Every subsequent daily PR then self-applies.
2. **Merge one of these PRs manually.** A single human merge backfills
   the fix and resets the clock.

If neither happens, tomorrow's `improve/2026-05-17` PR will say the
exact same thing.
