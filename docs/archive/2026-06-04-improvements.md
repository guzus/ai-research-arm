## Issue found — arXiv author extraction regressed to bare "et al."

The single concrete quality defect: in `research/arxiv/2026-06-03-papers.md`,
**every paper's `Authors` field is a bare `et al.`** — both the Top-5 highlights
and the by-category tables. The same regression appears in
`2026-06-01-papers.md`.

This is a regression, not a baseline limitation: older outputs
(`2026-05-17` through `2026-05-26`, and `2026-06-01`'s highlights) carry real
author lists with affiliations, e.g.
*"Kian Kenyon-Dean, Alina Selega, … Oren Kraus (Recursion)"*. The data is
available — the `arxiv-mcp-server` MCP returns a full `authors` field per paper
(confirmed both by the older outputs and by the server's documented metadata
schema) — so the loss of names is a prompt-discipline failure, not missing data.

### Root cause

`.github/workflows/daily-arxiv.yml` modelled the field as `**Authors**: [names]`
with no instruction forbidding a bare `et al.`, so the agent drifted to the
lowest-effort placeholder. The template itself showed `[names]`, which does not
push the agent to actually pull names from the MCP metadata.

### Fix

> **Delivery note:** the automation token used for this PR lacks the GitHub
> App `workflows` permission, so it cannot push edits to `.github/workflows/`.
> The workflow change is therefore shipped as an applyable patch,
> [`2026-06-04-daily-arxiv.patch`](2026-06-04-daily-arxiv.patch), to be applied
> by a maintainer (or by the `daily-improve` workflow, which does hold the
> permission) with:
>
> ```bash
> git apply docs/archive/2026-06-04-daily-arxiv.patch
> ```

The patch edits `daily-arxiv.yml` only (prompt-level, zero runtime/cost change):

1. Added an explicit **"AUTHORS — REQUIRED, no bare 'et al.'"** instruction:
   surface the first author's real name, append `, et al. (affiliation)` when
   there are more, and pull names from the MCP `authors` field. A bare `et al.`
   is rejected; `(authors unavailable)` is allowed only when the metadata
   genuinely lacks authors, and such papers should not be featured in the Top 5.
2. Updated the FORMAT template line from `[names]` to an explicit
   `[first author full name, et al. (affiliation) — real names, never a bare "et al."]`
   so the template no longer models the degenerate output.
3. Tightened the SKIP rules: removed *"Papers from unknown authors with few
   citations"* as a skip trigger, because 24-hour-old papers have ~zero
   citations by definition and that rule risked silently dropping exactly the
   fresh papers the lane exists to surface. Replaced with explicit guidance to
   judge on novelty/substance, not citation count.
